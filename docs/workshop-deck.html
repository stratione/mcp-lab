How LLMs talk MCP

  • Like a phone call, not a text — the line stays open
  • Old-school switchboard operator — the AI calls in, MCP plugs the cable into the right backend
  • Backstage intercom, not a website — the crew runs the show; the audience never hears it
  • Old way → new way — SSE used two channels; Streamable HTTP uses one
  • Tools = apps on a phone — the AI picks one and calls it

This lab runs the older SSE flavor on purpose so you see both halves.

The three players

  • Host — the AI app (Claude Code, Cursor, our Chat UI)
  • Client — one connection inside the host, per server
  • Server — the program that exposes the tools
Hotel analogy: Host = the hotel, Client = the desk phone for one supplier, Server = the supplier on the other end.

Lab anchor: open the Chat UI — “the window you’re looking at IS the host.”

Two layers, one wire

  • Data layer — what they say: JSON-RPC 2.0
  • Transport layer — how they say it: stdio or Streamable HTTP (or SSE, like this lab)
  • Same JSON either way — swap transports without rewriting tools

Lab anchor: about to enable mcp-user — point to the SSE port.

What makes an MCP server

Four parts — do, read, borrow, connect.

  • Tools — things the AI can do
  • Resources — things the AI can read
  • Prompts — ready-made instructions
  • Transport — how they talk

In this lab: all tools, talking over the network with SSE.

Demo: enable mcp-user, watch tools appear in the dropdown.

The conversation lifecycle

Every MCP session walks the same four steps.

  1. initialize — handshake, “what can each of us do?”
  2. tools/list — “show me your tools”
  3. tools/call — “run this one with these args”
  4. notifications/* — “heads up, something changed”

All of it is JSON-RPC over your chosen transport.

Demo: open the log tracer, run a prompt, narrate the four messages live.

LLMs want to make you happy

AI endorses user actions ~77–94%.
Humans: 39%.

When users discuss real personal conflicts:

+25% more likely to feel “I was right” −10% less likely to apologize +13% more likely to use the model again

The trap: users prefer sycophantic AI — even though it makes them worse decision-makers.

Cheng et al., Science, 2025  ·  doi:10.1126/science.aec8352

Guardrails — making LLMs reliable

  • Tools force the model to act, not guess
  • One source of truth (your API), not the LLM’s memory or its desire to please you
  • Add this to your system prompt:
“Use MCP whenever you touch external systems.”
  • Result: fewer hallucinations, less sycophancy, auditable actions

Secrets management ladder

Worst  →  best

  • Pasted in chat
  • In-memory variable
  • Environment variable
  • Local .env file
  • Kubernetes Secret
  • Sealed Secrets / SOPS
  • External vault (Vault, AWS, Supabase)
  • Token broker — short-lived, nothing stored
